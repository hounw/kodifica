# Project Journal

Memoria. Leer antes:

```bash
tail -n 80 project-journal.md
```

Reglas: entrada despues de cada commit. Corto. No diff. No secretos.

## Formato

```md
## YYYY-MM-DD - tipo: resumen

Commit: [hash|pendiente]
Cambio:
- 
Razon:
- 
Decision:
- 
Impacto:
- Producto:
- Tecnico:
- Operacion:
Pendiente:
- 
```

## Entradas

## YYYY-MM-DD - docs: inicio

Commit: pendiente
Cambio:
- Creados docs vivos.
Razon:
- Separar producto, tecnica, memoria.
Decision:
- Producto: `product-definition.md`.
- Tecnica: `technical-definition.md`.
- Memoria: `project-journal.md`.
Impacto:
- Agente lee tail antes de trabajar.
Pendiente:
- Poner hash primer commit.
